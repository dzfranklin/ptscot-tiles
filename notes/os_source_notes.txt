Open Rivers:
    https://docs.os.uk/os-downloads/networks/os-open-rivers
    Only has lines whereas vmdvec has polys for wider rivers
    Lines are precision than vmdvec
    Has names as line properties (vmdvec has name points where a 1:25k map should display)
    Alternate names seem to be names of specific bits
        E.g. name: "River Spey", alternate: "Junction Pool"

Open Names:
    Includes far more names than vmdvec. Does not include river names

Terrain 50:
    The Terrain 50 contours weren't generated off the Terrain 50 grid. I think they were generated by
    smoothing Terrain 5 contours. They seem to be the same used in Landranger.

    Compared to say Copernicus 30m Terrain 50 has been manually and automatically cleaned and smoothed.
    For example it does not have field boundaries.

VectorMap District (vmdvec)
    Older system of representation, has some data Open ZoomStack lacks
    Has less data than VectorMap local but I think just as precise

Open ZoomStack:
    https://docs.os.uk/os-downloads/contextual-or-derived-mapping/os-open-zoomstack
    Very similar to VectorMap district, but
        - has contours from Terrain 50
        - has national_parks
        - has urban_areas
        - has local_buildings (a bit fewer than OSM though)
        - has sites (eg school grounds)
        - has greenspace (for parks it's similar to OSM)
        - no spotheight (vmdvec has spotheight from Terrain 50)
        - no roundabout
        - represents tunnels differently, maybe less detail for roads but didn't bother checking
        - no motorway_junction
        - no glasshouse
        - functional_site is subsumed within names, looses categories like education
        - names: no styling data. a few small differences eg no Forest of Mar

Leisure (Landranger/Explorer):
    Up to 1,904,762 free map views (i.e. 15 tiles) per month

Vector Tiles:
    Like NGD is 60x more expensive than Leisure
    Has field boundaries, cairns, etc.

NGD:
    NGD is 60 TIMES more expensive than the equivalent Explorer raster maps, so probably not worth it

    Tiles overview: https://docs.os.uk/osngd/getting-started/access-the-os-ngd-api/os-ngd-api-tiles/what-data-is-available
    Features overview: https://docs.os.uk/osngd/getting-started/access-the-os-ngd-api/os-ngd-api-features

    Features are six times more expensive for a query of up to 100 than 4 tiles (~1 map view). So even though I only
    want a few feature types I think the map version makes more sense.

    The sample map is only visibly better than open data on zoom>16

    NGD's demo style pulls contours, ornaments from Open ZoomStack

    paths: Not worth it. Missing many paths that are on OS (eg see 55.879019, -3.214971)

    ** field boundaries **
        https://docs.os.uk/osngd/using-os-ngd-data/os-ngd-structures https://docs.os.uk/osngd/data-structure/structures/structure-features/field-boundary

    ** cairns **
        { "id" : "OS/NGD/str_fts_structure/Cairn", "type" : "fill", "source" : "ngd-base", "source-layer" : "str_fts_structure", "filter" : [ "==", "_symbol", 19 ], "minzoom" : 16, "maxzoom" : 20, "layout" : { }, "paint" : { "fill-color" : "#E7C9C8" } }

    detailed cliff edges, ridge lines, etc
        but I think the stylized ornaments read better on a map for my purposes
        https://docs.os.uk/osngd/data-structure/land/land-features/landform

    water
        https://docs.os.uk/osngd/data-structure/water/water-network
        slightly higher resolution streams than Explorer

    landcover: higher resolution even than Explorer, is that really worth the clutter?

    names:
        https://docs.os.uk/osngd/data-structure/geographical-names
        Don't see much difference compared to VMD.
        Crowd sourced name points are cool but probably not worth it?
