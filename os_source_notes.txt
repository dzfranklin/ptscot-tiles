Terrain 50
    The Terrain 50 contours weren't generated off the Terrain 50 grid. I think they were generated by
    smoothing Terrain 5 contours. They seem to be the same used in Landranger.

    Compared to say Copernicus 30m Terrain 50 has been manually and automatically cleaned and smoothed.
    For example it does not have field boundaries.

Leisure (Landranger/Explorer):
    Up to 1,904,762 free map views (i.e. 15 tiles) per month

Vector Tiles:
    Like NGD is 60x more expensive than Leisure
    Has field boundaries, cairns, etc.

NGD:
    NGD is 60 TIMES more expensive than the equivalent Explorer raster maps, so probably not worth it

    Tiles overview: https://docs.os.uk/osngd/getting-started/access-the-os-ngd-api/os-ngd-api-tiles/what-data-is-available
    Features overview: https://docs.os.uk/osngd/getting-started/access-the-os-ngd-api/os-ngd-api-features

    Features are six times more expensive for a query of up to 100 than 4 tiles (~1 map view). So even though I only
    want a few feature types I think the map version makes more sense.

    The sample map is only visibly better than open data on zoom>16

    NGD's demo style pulls contours, ornaments from Open ZoomStack

    paths: Not worth it. Missing many paths that are on OS (eg see 55.879019, -3.214971)
